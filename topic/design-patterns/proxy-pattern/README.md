### 代理模式

**为其它对象提供一种代理以控制这个对象的访问。在某些情况下，一个对象不适合或者不能直接引用另一个对象，而代理对象可以在客户端和目标对象之间起中介作用**

#### 虚拟代理

**虚拟代理是把一些开销很大的对象，延迟到真正需要去执行的时候才去创建执行**

#### 缺点

**增加系统的复杂度**

#### 优点

**把代理对象和被调用对象分离，降低系统的耦合度，在客户端和目标对象之间起一个中介作用，保护目标对象**
